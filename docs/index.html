"Mechanistic interpretability is the only path to verifiable AI compliance."

Every other compliance tool treats the model as a black box and documents inputs and outputs. Annex IV §2(d) demands more. It requires documentation of the internal reasoning structure — which means you need circuit-level analysis.

Glassbox was built to solve exactly this problem. We use sparse autoencoders to decompose attention heads into interpretable features, then map those features to the exact language of the EU AI Act. The result is documentation no human auditor could produce manually in any reasonable timeframe.

FAQ

Frequently asked questions

Is Glassbox ready for production use? +
Yes. v2.0.0 is stable and available on PyPI. It ships with a full test suite, comprehensive documentation, and has been validated against the EU AI Act text. The arXiv paper (2603.09988) describes the methodology in peer-reviewed detail. Teams have already used it to produce Annex IV documentation accepted in pilot audits.
Which models does Glassbox support? +
Glassbox currently supports transformer-based LLMs with accessible attention weights — including GPT-2 family, Llama family, Mistral, Gemma, and Phi. Support for additional architectures is expanding. The circuit tracing methodology is architecture-agnostic; what varies is the SAE training configuration per model family.
How does the circuit tracing actually work? +
We train Sparse Autoencoders (SAEs) on attention layer activations to decompose them into a dictionary of interpretable features. Each feature corresponds to a detectable pattern in the model's internal reasoning. We then map these features to circuit diagrams showing which heads collaborate on which reasoning steps. Full methodology in arXiv:2603.09988.
Is the August 2026 deadline legally binding? +
Yes. EU AI Act Article 113 specifies that provisions for high-risk AI systems (including those covered by Annex IV documentation requirements) apply from August 2, 2026. Non-compliance exposes operators to penalties of up to €30 million or 6% of annual global turnover, whichever is higher. There is no grace period announced for Annex IV documentation.
Can I self-host the entire stack? +
Yes. The core library is MIT licensed — install it, inspect the source, modify it, host it wherever you need. The BSL 1.1 commercial license applies to production SaaS deployments serving third parties. Solo operators and teams using Glassbox for their own compliance documentation can use the MIT core freely.
